发表于 2025-12-02 阅读次数： 本文字数： 3.1k 阅读时长 ≈ 3 分钟

flutter

安装

首先查看下报错信息：

报错信息为：连接不上

  • pub.dev

  • storage.googleapis.com

  • maven.google.com

  • cocoapods.org

  • github.com

环境为：Win11+Clash Verge全局+系统代理

解决流程

首先使用浏览器查看，发现以上页面都能打开。

感觉可能是终端没走代理的原因，就把代理调整为TUN模式

使用终端curl github，成功返回页面

使用终端curl https://maven.google.com/com/
→ 404 That’s an error.（但成功返回页面）

AI的解释为

因为 Flutter/Gradle 访问 maven.google.com 不是访问 /com/
它访问的是：

并且这些文件需要：

  • TLS 1.2 / TLS 1.3
  • 正确的 CA 证书链
  • 完整的网络握手

你之前出现的错误：

这说明 TLS 握手阶段就失败了
而 curl 能成功 → 说明：

  • 你使用的 curl 内置证书是 OK 的
  • Java / Flutter / Dart 使用的证书链不 OK

❗ 这类问题绝大多数都是 Windows 环境里的 CA 证书缺失/损坏 或 代理/杀软注入了假证书（MITM）

然后又去检查CA证书

我看到了堡塔……然后把堡塔的证书删掉再次尝试，发现成功

结束

早知道就不拿电脑瞎折腾了，当初想试一下win安装堡塔来着QAQ
